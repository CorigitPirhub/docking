# 问题定义

**“多车对接”问题定义**

**1. 系统基本设定 (定义0)**

- **初始状态**：多辆同构的阿克曼模型车辆在环境中处于自由运动状态。
- **车辆模型**：所有车辆均为标准阿克曼转向底盘。
  - **几何参数**：车长 $L_{car}=1.0$ 米，车宽 $W_{car}=0.4$ 米，轴距 $L_{wheelbase}=0.7$ 米。
  - **铰接配置**：首车（**头车**）仅在尾部装备铰接点；其余车辆头部和尾部均装备铰接点。铰接点伸出长度均为 $L_{hitch}=0.3$ 米。车辆头部铰接点处安装前向视觉摄像头。
  - **运动学参数**：单车最小转弯半径 $R_{min\_single}=1.0$ 米，对应最大前轮转向角 $\delta_{max}=35^\circ$，最大转向角速率 $\dot{\delta}_{max}=30^\circ/s$。
- **感知与定位**：
  - **全局定位**：每辆车配备卫星定位模块，位置偏差标准差 $\sigma_{pos\_global}=0.05$ 米，航向偏差标准差 $\sigma_{\psi\_global}=3^\circ$，更新频率 $10\text{Hz}$。
  - **视觉感知**：车辆头部摄像头视场角 $120^\circ$，有效感知距离 $d_{vis\_max}=1.5$ 米。在有效范围内，相对位姿误差标准差为：位置 $\sigma_{pos\_vis}=0.01$ 米，航向 $\sigma_{\psi\_vis}=1^\circ$，帧率 $30\text{Hz}$。
  - **全局感知假设**：系统假设每辆车均拥有完备的全局环境感知信息，即共享同一张包含所有静态障碍物与边界的全局地图，并可通过通信（忽略延迟）实时获知所有其他车辆的全局位姿、轮速及转角信息。
- **头车独立任务**：头车拥有独立的全局导航任务（起点A至终点B）。头车最大线速度 $v_{max}=2.0$ m/s，最大线加速度 $a_{max}=1.0$ m/s²。**头车优先完成自身任务，其运动不受后车对接影响**，其规划轨迹对全系统共享。

**2. 核心问题：两车对接过程 (定义1)**

后车对接前车被建模为一个**两阶段视觉伺服控制问题**：

- **第一阶段：全局趋近阶段** ($d > 1.5m$)
  - 基于共享的全局定位信息，执行到动态目标点（前车尾部对接点）的实时运动规划与跟踪。
- **第二阶段：视觉伺服阶段** ($d \le 1.5m$)
  - 基于自身前向摄像头获取的高精度相对位姿信息，对前车尾部对接点执行视觉伺服闭环控制。

**过程约束**：

1. **运动与规划约束**：车辆运动需满足阿克曼模型、最小转弯半径（$1.0$ 米）、禁止原地转向等约束。速度/加速度限制：$v \le 2.0$ m/s, $a \le 1.0$ m/s², 减速度 $a_{dec} \le 1.5$ m/s², 横摆角速度 $\omega \le 2.0$ rad/s。每辆车需具备实时局部运动规划能力。
2. **安全约束**：规划与执行过程中必须进行实时碰撞检测，确保与静态障碍物、其他车辆不发生碰撞。车辆与障碍物边界最小净空应 $\ge 0.1$ 米。检测需考虑车辆的实际几何包络（含铰接点）。
3. **末端约束**：对接完成的判定标准为：位置误差 $\epsilon_p < 0.02$ 米，航向误差 $\epsilon_\psi < 5^\circ$，速度差 $\epsilon_v < 0.1$ m/s，且需保持 $\tau_{hold}=0.5$ 秒。对接瞬间两车车身应保持平直。
4. **感知切换约束**：禁止在 $1.5$ 米处对目标点感知进行硬切换。需在过渡区（$d \in [1.2m, 1.8m]$）设计**加权融合算法**（如 $w_{vis} = (1.8 - d) / 0.6$），以避免控制指令跳变。
5. **感知异常处理**：若在视觉伺服阶段目标脱离摄像头视场，系统需具备回退至全局定位模式或触发重规划的能力。
6. **单车规划与动态避障约束**：每个自由运动的车辆都必须具备**实时局部运动规划**能力，规划无碰撞路径。规划器必须考虑自身运动学约束、实时环境感知、动态避障（躲避其他自由车辆）以及目标车辆避让。

**3. 扩展框架：多车对接问题分解 (定义2)**

- **全局优化目标**：最小化整体对接完成时间 $T_{done}$，且 $T_{done} \le T_{max}=60$ 秒。
- **任务可完成性判定**：估算完成剩余对接所需最短时间 $T_{est}$。判定条件：若 $n_{dock} \times T_{dock\_est} + T_{margin} > T_{leader\_remaining}$（其中 $T_{dock\_est}=10$ 秒, $T_{margin}=5$ 秒），则判定任务不可完成。
- **维度一：受限序列决策**
  - **任务**：为每辆自由车动态选择最优对接目标。
  - **核心约束**：决策必须引入几何可达性预判，过滤会导致死锁的序列。
- **维度二：协同运动规划与控制**
  - **任务本质**：执行两车对接，适应多车协同上下文。
  - **前车模态适应性**：辨识前车是单车还是列车（列车需考虑铰接动力学）。
  - **预测性主动对接**：利用前车未来轨迹信息，规划主动截获路径。
  - **前车配合行为**：近距离对接阶段，前车执行“主动配合策略”（减速、稳向）。
  - **反应式重规划**：突发障碍或预测失效时实时触发重规划。
- **维度三：链式系统安全与整体规划**
  - **系统形态演化**：对接成功后，连接体演变为**铰接列车**。列车总长 $L_{train}(n) = 1.0 + (n-1) \times 1.6$ 米。
  - **列车运动学**：列车最小转弯半径 $R_{min\_train}(n) = R_{min\_single} + 0.5(n-1)$ 米。最大铰接角（防折刀约束）$\phi_{max}=45^\circ$。
  - **列车整体规划能力**：列车作为整体规划，由头车执行**列车整体规划器**，生成满足列车运动学约束的统一轨迹。
  - **规划模式切换**：单车对接成功后，切换至“列车整体规划跟随模式”。
  - **核心安全约束**：实现“扫掠守护”机制，将中间车辆碰撞风险转化为头车控制输入约束 $\mathcal{U}_{safe}$。

**4. 环境、评估与实现要求 (定义3)**

- **环境表征**：系统运行于二维栅格地图，分辨率 $0.05$ 米，尺寸建议 $20m \times 40m$ 或根据任务调整。仅含静态障碍物。
- **评估指标**：
  1. **任务级**：整体对接成功率 $\ge 95\%$，总耗时 $T_{done} \le 60s$，任务可完成性判定准确率 $\ge 90\%$。
  2. **单车级**：单次对接最终位置误差 $\le 0.02m$，航向误差 $\le 5^\circ$，单次对接耗时 $\le 15s$。
  3. **过程质量**：线加速度变化率 $\le 5 m/s^3$，角加速度变化率 $\le 10 rad/s^3$；感知切换平滑度满足约束。
  4. **安全性**：碰撞次数 $0$，最小净空 $\ge 0.1m$。
- **实现要求**：
  1. **核心算法自主实现**：针对感知平滑切换、可达性预判、扫掠守护等难点自主设计算法。
  2. **系统架构**：采用“感知-决策-控制”分层架构。
  3. **实时性**：控制主循环 $\ge 10\text{Hz}$，高层调度 $0.5\text{-}1\text{Hz}$，中层规划 $5\text{-}10\text{Hz}$。
  4. **可视化**：生成高清动画，展示轨迹、FOV、安全边界、对接状态、传感器切换状态等。
