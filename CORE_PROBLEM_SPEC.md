# 动态可重组多智能体系统核心问题规格（执行版）

## 1. 文档目的与适用范围

本规格用于明确“动态可重组多智能体系统的时空资源协同优化与自适应编队规划”问题的任务目标、决策原则、分层接口边界与验证出口。  
本规格面向后续工程实现与实验评估，要求描述可执行、可验证、可复用，且与已有文档保持一致：

- `problem_definition.md`：车辆/对接/安全等物理与基础控制约束。
- `core_requirements.md`：统一数学符号与优化目标。
- `SCENARIO_CLASSIFICATION.md`：场景分类、标签与标准化测试入口。

本规格默认研究范围为：二维静态障碍环境、全局信息共享、忽略通信时延，不包含动态障碍。

## 2. 任务描述（无歧义重述）

### 2.1 初始条件与总体目标

初始时，多辆同构阿克曼车辆分散在场景中，需从起始区域前往目标区域执行运输任务。系统总目标是联合最小化：

1. 任务总耗时（到达目标并满足拓扑合法的完成时间）。
2. 系统总能耗（考虑编组状态的节能收益）。

### 2.2 已知先验与工程含义

1. 依据文献《Internal-Stably Energy-Saving Cooperative Control of Articulated Wheeled Robot with Distributed Drive Units》，列车状态相较单车状态可降低约 3%–5% 能耗。  
工程含义：当环境允许且对接可行时，应优先推动更多车辆完成对接并以列车状态行驶，以获得节能收益。

2. 头车默认策略为“非等待直达终点”。  
工程含义：头车原则上不主动停车等待后车，而是持续规划并执行前往终点的轨迹，并将预测轨迹共享给后车。  
补充边界：允许策略层在少数场景下决策“短时等待”，但该行为必须通过代价函数证明其对全局目标（时间+能耗）更优，而非默认动作。

3. 对接序列必须通过显式代价函数决策。  
工程含义：需要建立统一评价函数，比较“谁先对接、谁后对接、是否并发对接、是否放弃对接”的优劣，避免基于规则堆砌。

4. 后车执行“预测性主动截获”，而非仅被动追尾。  
工程含义：后车可基于前车未来轨迹选择截获点（前车路径上的未来可达点）进行对接，而非固定追踪当前尾部点。

5. 核心创新目标是“策略层与规控层解耦”。  
工程含义：先建立平台无关的策略决策层，再通过清晰接口将策略命令下发至具体平台规控层；平台侧负责可行执行与反馈。

6. 场景决定可编组上限，策略需动态重构。  
工程含义：系统需根据环境约束决定最大可通过编组规模，并支持“先对接、后解编、再对接”等混合过程，而非单一长期形态。

## 3. 问题边界与默认假设

1. 障碍仅为静态固定障碍。
2. 所有车辆共享全局地图与全局状态信息。
3. 车辆动力学、对接门限、安全约束、防折刀约束遵循 `problem_definition.md`。
4. 场景分类与标签计算遵循 `SCENARIO_CLASSIFICATION.md`。
5. 目标函数形式、变量符号与约束集合遵循 `core_requirements.md`。

## 4. 核心问题定义（策略层视角）

核心问题：在满足运动学与安全约束前提下，使系统智能决定“何时对接、何时独立行驶、何时解编、是否等待”，并最小化时间-能耗联合代价。

### 4.1 分层解耦架构（核心创新点）

1. 策略层（平台无关）  
输入：全局态势、拓扑状态、场景标签、可行性反馈。  
输出：离散决策变量与指令集合 \(z^k\)（对接/解编/等待/任务分配）。  
责任：仅做“做什么、何时做、对谁做”的决策，不直接输出底层控制量。

2. 规控层（平台相关）  
输入：策略层指令、当前物理状态。  
输出：轨迹/控制执行结果、可行性与异常反馈。  
责任：负责“怎么做”，包括路径规划、轨迹跟踪、安全滤波、对接控制。

3. 关键接口要求  
- 指令应标准化（例如 `DockingCommand`、`SplitCommand`、`WaitCommand`）。  
- 支持异步多率运行与事件回执（成功、失败、过期、冲突、撤销）。  
- 保证时空一致性：策略层决策基于可确认的最新状态与反馈闭环。

### 4.2 基于时间-能耗的对接序列决策

目标：在初始分散状态下求解最优或近优对接序列。  
代价函数应至少包含：

1. 追击与截获时间成本（后车追上目标编组的时间）。
2. 对接执行时间与风险成本（失败重试、姿态调整代价）。
3. 节能收益项（编组后能耗下降收益）。
4. 对全局完成时间的影响（含潜在等待代价）。

默认行为：

1. 头车“不停车等待”为默认基线策略。
2. 后车执行“主动截获”而非纯尾随。

可选算法方向：

1. 分布式拍卖/竞价分配。
2. 滚动时域优化（MPC）或混合整数近似优化。

### 4.3 动态通行判定与自适应解编

目标：在路径受限区域（如瓶颈/窄弯）前，动态决策保留编组或触发解编。

决策依据：

1. 环境可通过能力 \(W_{env}\)。
2. 编组需求宽度 \(W_{req}(n,\kappa)\)（显式考虑曲率与扫掠影响）。
3. 最大可通过编组规模 \(N_{max\_pass}\)。

策略规则：

1. 若当前编组满足通行且继续编组有收益，鼓励保持/增加编组。
2. 若不满足通行条件，必须在约束区入口前完成解编。
3. 解编点选择需平衡“尽量晚解编以保节能”和“预留机动空间保安全”。

### 4.4 预测性主动对接规划

定义：截获点是“前车（或目标列车头车）共享轨迹上的未来路径点”，而非任意空间静态点。  
目标：为后车生成满足运动学与安全约束的平滑截获轨迹，并在合适窗口完成对接。

执行要求：

1. 使用前车未来轨迹预测（窗口 \(T_{pred}\)）。
2. 联合约束静态障碍与其他自由车避让。
3. 对接阶段支持异常回退（视觉丢失、路径被阻塞、目标不可达）。

### 4.5 任务可完成性判定与异常回退

目标：在运行时及时识别“继续对接是否仍然有意义”，并防止无效追击。

规则：

1. 估算剩余对接完成时间 \(T_{est}\)。
2. 估算头车到终点剩余时间 \(T_{leader\_remain}\)。
3. 若 \(T_{est} > T_{leader\_remain}\)（可加安全裕量），则判定当前对接任务不可完成，立即切换为独立行驶或备用任务。

异常处理：

1. 视觉伺服期间丢失目标：回退到全局模式并重规划。
2. 路径阻塞或可行性翻转：触发事件重规划或安全停车。

### 4.6 场景生成与验证体系

目标：建立标准化场景集，验证策略在不同结构约束下的泛化能力。

场景类型：

1. Type A（全开放）：验证全对接策略收益上界。
2. Type B（瓶颈约束）：验证编组规模受限与解编决策。
3. Type C（混合复杂）：验证动态重组闭环（先合后分/先分后合/多次切换）。

评估出口：

1. 成功率与鲁棒性（完成任务、无碰撞、无死锁）。
2. 总耗时与总能耗联合评估（帕累托前沿或权重扫描）。
3. 重构次数、等待时间、失败回退率等过程指标。

## 5. 决策优先级（冲突消解顺序）

当多个目标冲突时，按以下优先级决策：

1. 安全与可行性约束（碰撞、防折刀、动力学可执行）。
2. 任务可完成性（避免无效对接导致任务失败）。
3. 总耗时最优（优先保障任务时限与整体完成时间）。
4. 总能耗最优（在满足前述条件下最大化编组节能收益）。
5. 过程质量优化（平滑性、重构次数、舒适性等次级目标）。

## 6. 交付要求（面向后续实现）

后续“核心问题实现”应至少给出以下可检验交付物：

1. 策略层决策模块（含输入输出协议与事件机制）。
2. 可复现实验脚本（Type A/B/C 标准场景批量运行）。
3. 指标统计与对比报告（时间、能耗、安全、重构行为）。
4. 失败案例归因（不可达、判定翻转、接口冲突、控制不可行）。
