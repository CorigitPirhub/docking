# 进阶基础支撑：场景分类（用于核心需求阶段）

## 1. 目标与约束
- 目标：为“动态可重组多智能体系统的时空资源协同优化”提供可复用、可量化、可对比的场景分类基准。
- 约束：仅考虑二维静态障碍场景；全局地图与全车状态可共享；不引入动态障碍。
- 地图基线：`40m x 20m`（可扩展），分辨率 `0.05m`。
- 分类语义约束：场景分类必须绑定运输任务（起点A到终点B），脱离任务路径的几何分类无效。
- 说明：本文档主要服务于 P2-P6 的多车重构/通行场景；P-1 两车 docking 闭环 benchmark 另行采用 `DOCK_SCENARIO_STANDARD.md` 中的 `DockBench-v1` 标准，以避免“多车运输分类”和“两车 docking 机制分类”混淆。

## 1.1 任务关键路径（Task-Critical Path）前置

- 在执行 Type A/B/C 分类前，必须先生成一条或多条从 `A -> B` 的候选全局路径 `Pi = {pi_1, pi_2, ...}`。
- 所有核心几何分析均以候选路径为中心进行，而非全图平均：
  - `N_max_pass_profile`
  - `dock_friendly_zones`
  - `split_mandatory_zones`
  - `reconfig_window`
- 对每条候选路径定义横向分析走廊 `C(pi, d_corridor)`，仅在该走廊内做通行能力与可重组分析。
- 若不同候选路径对应不同最优重组策略，则场景应标注为“路径依赖型”（默认归入 Type C 的高复杂实例）。
- 基础支撑阶段的简化规则：
  - 默认仅采用单条主路径 `pi*` 做分类（推荐“最短路径”或“最速路径”）；
  - 多候选路径冲突分析作为后续进阶能力，不纳入当前最低实现闭环。

## 2. 一级分类（与策略目标强绑定）

### Type A：开放协同型（鼓励尽快对接）
- 定义：不存在强制单车通行瓶颈，列车形态在主要通道可持续通行。
- 策略目标：最大化对接比例，最小化总能耗。
- 代表子类：
  - A1 `Open-Plain`：近似空旷，无关键瓶颈。
  - A2 `Open-Sparse`：少量稀疏障碍，绕行代价低。
  - A3 `Open-WideS`：宽 S 弯，单车/列车均可通过（判定需包含转弯扫掠包络与曲率约束）。

### Type B：瓶颈约束型（必须解编或限制编组规模）
- 定义：存在关键瓶颈区，通行宽度限制决定最大可编组规模 `N_max_pass`（注意：最严格的情况也应当允许一辆单车可通行，否则无解无意义）。
- 策略目标：在瓶颈前自适应解编；通过后视情况重组。
- 代表子类：
  - B1 `Bottleneck-SingleOnly`：仅单车可通过，`N_max_pass=1`。
  - B2 `Bottleneck-KCap`：最多 `K` 车编组可通过（如 `K=2/3/4`）。
  - B3 `Bottleneck-Serial`：多级瓶颈串联（如先 `K=3` 后 `K=1`）。
  - 注：`N_max_pass` 不能只基于通道几何宽度，还必须包含局部弯道曲率导致的扫掠体放大效应（单车和列车最小转弯半径的差异是关键，要格外注意列车的最小转弯半径公式，见 problem_definition.md 的列车运动学）。

### Type C：混合重构型（先合后分/先分后合）
- 定义：同一任务中同时存在“开放区 + 瓶颈区 + 再开放区”或多约束组合。
- 策略目标：时空协同最优（时间/能耗权衡），动态切换对接与解编。
- 代表子类：
  - C1 `Hybrid-OpenNarrowOpen`：开放-窄道-开放。
  - C2 `Hybrid-MultiPocket`：多个可临时集结/重构区域。
  - C3 `Hybrid-OcclusionDock`：可见性受限对接区 + 瓶颈。

## 3. 二级标签（场景元数据，供策略层直接使用）

每个场景除 `type` 外，必须标注以下元信息：

- `bottleneck_count`：瓶颈数量。
- `N_max_pass_global`：全局最严格瓶颈允许的最大编组规模。
- `N_max_pass_profile`：沿主路径分段的允许编组规模曲线。
- `open_area_ratio`：开放区域占比（面积）。
- `detour_factor`：绕行系数（最短无障碍路长 / 直线距离）。
- `dock_friendly_zones`：适宜对接区域集合（宽阔、低障碍、可见性好、低曲率/高直线度）。
- `split_mandatory_zones`：必须完成解编的区域集合。
- `occlusion_level`：对接可见性等级（低/中/高）。
- `reconfig_window`：可重组窗口（从约束区退出到下一关键点之间，允许追赶并完成对接的距离/时间余量）。
- `decision_horizon`：决策预见性需求（低/中/高），衡量需要提前多远做解编/对接决策。
- `coordination_intensity`：协同强度（低/中/高），衡量车辆汇合/分离与时序耦合频率。
- `initial_dispersion_mode`：车辆初始分散模式（`Clustered_At_A` / `Random_Scattered` / `Uniform_Spread`）。
- `initial_state_seed`：初始状态采样随机种子（用于复现实验）。

## 4. 几何判定标准（分类落地规则）

- 对每段候选通道，通行能力由“宽度约束 + 曲率约束”共同决定：
  - 宽度约束：`N_w = f(W_env)`，基于扫掠包络宽度判定可通过最大编组；
  - 曲率约束：`N_k = g(kappa)`，基于局部曲率 `kappa` 与列车最小转弯半径/扫掠体判定可通过最大编组；
  - 合成判定：`N_max_pass = min(N_w, N_k)`。
- 解释：同样通道宽度下，`90 deg` 急弯可能因扫掠包络放大导致编组不可通过，即使直道可通过。
- `N_max_pass` 定义为关键路段最严格约束下仍可行的最大 `n`。
- 若场景所有关键通道均满足 `N_max_pass >= N_total`，归类为 Type A。
- 若存在关键通道 `N_max_pass < N_total` 且结构单一，归类为 Type B。
- 若同场景内 `N_max_pass_profile` 随空间显著变化并触发多次重构，归类为 Type C。

### 4.1 对接友好区（`dock_friendly_zones`）判定补充

- 区域需同时满足以下条件才可标记为“适宜对接区”：
  - 净空充足：局部最小净空 `clearance >= c_dock_min`；
  - 障碍稀疏：局部障碍密度 `rho_obs <= rho_dock_max`；
  - 可见性良好：无遮挡视线比例 `los_ratio >= los_min`；
  - 直线度约束：局部曲率满足 `|kappa| <= kappa_dock_max`（必要条件）。
- 说明：即使宽度足够，若处于强弯区域，视觉伺服末端易触发较大铰接角，增加折刀风险；因此低曲率是对接区硬门槛。

## 5. 基准场景集（建议最小闭集）

为保证策略泛化，最少采用以下 9 类基准：

1. `A1` 空旷基准（全对接最优应显著成立）
2. `A2` 稀疏障碍（全对接仍应高概率最优）
3. `A3` 宽 S 弯（验证列车规划与跟踪一致性）
4. `B1` 单车瓶颈（必须解编）
5. `B2-K2` 双车瓶颈（限制编组规模）
6. `B3` 串联瓶颈（多次重构）
7. `C1` 开放-窄道-开放（先合后分后合）
8. `C2` 多口袋区（重构时机优化）
9. `C3` 遮挡对接混合（可见性异常 + 重构）

## 5.1 参数化生成规则（必须）

每个子类应由“参数化生成器”产生多个实例，而不是单张固定地图。

- 通用初始状态参数（适用于所有子类）：
  - `initial_dispersion_mode`：
    - `Clustered_At_A`：车辆在起点区域 A 附近聚集；
    - `Random_Scattered`：车辆在可行自由空间内随机分散；
    - `Uniform_Spread`：车辆按规则网格/均匀策略分布于任务相关区域。
  - `initial_pose_jitter`：初始位置/航向扰动幅度；
  - `initial_state_seed`：初始状态生成随机种子；
  - `spawn_clearance_min`：出生时与障碍/边界/其他车的最小净空。
- 说明：`initial_dispersion_mode` 会显著影响“初始协同耦合度”，应参与 `coordination_intensity` 标签判定。

- `A3 Open-WideS` 生成器参数：
  - `R_s`：S弯等效曲率半径；
  - `W_lane`：可通行带宽；
  - `L_open`：S弯前后开放区长度。
  - 生成后自动验证：`N_max_pass_global = N_total` 才可标注为 A3。

- `B2 Bottleneck-KCap` 生成器参数：
  - `K`：目标可通过最大编组规模（`K=2/3/4/...`）；
  - `W_neck`：瓶颈宽度；
  - `R_neck`：瓶颈弯道半径（可为 0 表示直道瓶颈）；
  - `L_buffer`：瓶颈前后机动缓冲区长度。
  - 生成后自动验证：`N_max_pass_global == K`（容差内）且单车可达。

- `C1/C2/C3` 生成器参数：
  - `segment_pattern`：开放/瓶颈/遮挡段的组合序列；
  - `L_reconfig`：可重组窗口长度；
  - `pocket_count`：临时集结区数量；
  - `occlusion_density`：遮挡密度。
  - 生成后自动验证：`N_max_pass_profile` 至少发生一次有效变化（触发重构需求）。

## 7. 验证出口（分类是否可用）

场景分类通过的最低条件：

1. 每个场景可自动产出 `type + 二级标签` 元数据。
2. `N_max_pass` 判定与仿真通行结果一致率 >= 95%。
3. 各类型至少有 1 个可视化样例（地图 + 关键区域标注）。
4. 分类可直接被策略层调用，不依赖底层控制器内部实现细节。
5. Type B/Bottleneck 生成时需执行“无解剔除”检查：至少保证单车可通过瓶颈；若瓶颈过窄导致单车不可达，则该场景在生成阶段直接剔除。

## 8. 实现建议：`N_k = g(kappa)` 的工程求解

- 难点：`g(kappa)` 与编组长度 `n`、扫掠包络、最小转弯半径耦合，通常难以写出稳定解析反函数。
- 建议工程实现：
  1. 离线查表：对 `n=1..N_max` 和曲率离散网格 `kappa_grid` 预计算可通行性或等效扫掠宽度，生成 `table[n, kappa_bin]`；
  2. 在线查询：给定局部 `kappa`，按插值/最近邻取 `kappa_bin`；
  3. 编组上限求解：对 `n` 做二分或倒序扫描，找到最大满足通行条件的 `n`，得到 `N_k`；
  4. 最终判定：`N_max_pass = min(N_w, N_k)`。
- 该方案优点：运行时复杂度低、稳定性高、易与策略层实时滚动优化对接。

## 9. 扫掠体模型建议（用于查表离线计算）

- 目标：在可计算成本下得到“安全保守”的 `g(kappa)` 估计。
- 建议模型：保守列车扫掠包络模型（Conservative Swept Envelope）
  1. 给定编组长度 `n`、曲率 `kappa`、铰接角约束 `phi_max`；
  2. 假设每节车厢沿头车轨迹的等距偏移线行驶（保守近似）；
  3. 对整列车在轨迹段上做离散前向仿真，收集所有车体外轮廓并取并集；
  4. 计算包络最小通行宽度需求 `W_req(n, kappa)` 与等效转弯占用。
- 模型输入必须显式包含：
  - 编组长度 `n`
  - 局部曲率 `kappa`
  - 防折刀约束 `phi_max`
  - 控制策略假设（后车跟踪律，建议固定为“后车轨迹跟随前车路径”的保守版本）
- 模型输出用于：
  - 构建 `table[n, kappa_bin]`
  - 计算 `N_k = g(kappa)`
  - 校验对接区可行性（避免在高曲率段触发铰接角风险）
